# Agents

An agent is a first-class platform resource with its own runtime configuration.

It combines a model, sandbox image, tools, permissions, and lifecycle policy.

## What an agent owns

- A name that users and operators can recognize.
- A sandbox environment where work is executed.
- Tool connections through MCP servers.
- Budget, access, and idle-timeout controls.

For deployment steps, see [Deploy your first agent](../start/deploy-your-first-agent.md).
